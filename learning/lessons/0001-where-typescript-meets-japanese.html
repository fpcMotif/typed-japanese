🌸 Typed Japanese · Lesson 1

Where TypeScript meets Japanese

Your mission: grow both at once. This lesson gives you the one idea that makes that possible — then a single first step on each side.

A Japanese verb is a type. Conjugating it is a type-level function. A grammar mistake is a type error the compiler catches before the code ever runs.

If that sentence makes sense by the end of this page, you've understood the heart of the whole project. Let's earn it.

1 Why these two fit together

It looks like a joke pairing — a programming language and a human language. But they share a backbone: strict rules for combining small pieces. Each side brings exactly what the other needs.

What's special about Japanese

Japanese conjugation is unusually systematic. Take a verb apart and it's almost mechanical: a fixed stem, plus an ending that changes by a clear rule depending on which form you want.¹ "to buy" runs:

Dictionary買う  kau
Te-form買って  katte
Past買った  katta
Negative買わない  kawanai

That is a function in disguise: give it (verb, form), get back a string. Where English has a pile of exceptions, Japanese has tables. Tables are easy to teach a computer.

What's special about TypeScript

Most languages' type systems only *check* your code. TypeScript's type system can also compute — it runs a small program at compile time, before anything executes. And crucially, it can compute on text: it can take string-shaped types apart and glue new ones together.² Three features make this possible, and you'll meet all three in §2.

The handshake

Put them together and the match is almost too neat:

Japanese ideaTypeScript tool
a specific word, e.g. 話すa literal type "話す"
a verb (stem + ending)an object type { stem; ending }
conjugation rulea type-level function
"that's the wrong form"a type error 🔴

Here is that last row, real and alive, from this repo:³

// 話す = "to speak": a godan verb, stem 話 + ending す
type 話す = GodanVerb & { stem: "話"; ending: "す" };

// Conjugate it into the polite ます形 form. Result is the literal type "話し".
type Result = ConjugateVerb<話す, "ます形">;

const ok:  Result = "話し";   // ✓ compiles — correct conjugation
const bad: Result = "話す";   // 🔴 Type error: "話す" is not "話し"
The compiler computed 話し on its own — you never wrote it. That's the type system running the grammar rule for you. Spell the conjugation wrong and your editor underlines it in red, instantly. That red squiggle is the whole pitch of Typed Japanese.

2 First step — TypeScript

The one skill the entire library is built on: a type-level function. Forget Japanese for sixty seconds and watch it on English, the way the author introduces it.⁴

A normal runtime function:

function conjugate(verb, form) {
  if (form === "past") return verb + "ed";
  if (form === "progressive") return verb + "ing";
  return verb;
}

The same logic, lifted into the type level:

type Conjugate<Verb extends string, Form extends EnglishForm> =
    Form extends "past"        ? `${Verb}ed`
  : Form extends "progressive" ? `${Verb}ing`
  : Verb;

type A = Conjugate<"walk", "past">;        // "walked"
type B = Conjugate<"talk", "progressive">; // "talking"

Three tools, and that's the whole toolkit for now:

1 · GenericsThe <Verb, Form> in angle brackets are the function's inputs.
2 · Conditional typesForm extends "past" ? … : … is type-level if/else.
3 · Template literals`${Verb}ed` glues strings together — at the type level.

Swap English's "ed"/"ing" for Japanese endings and you have exactly the real ConjugateVerb — just with more branches. You already understand its skeleton.

It's the literal type "jumping". The second branch matched, and `${"jump"}ing` glued to "jumping". No code ran — the compiler worked it out. That's a type-level function.

3 First step — Japanese

To feed a verb to that function, you must first see a verb the way the type system does: stem + ending, belonging to one of three groups.⁵

VerbMeansGroupStemEnding
話す hanasuto speakgodan 五段話す
食べる taberuto eatichidan 一段食べる
する suruto doirregular 不規則—whole word
  • Godan (五段) — "five-row." The most common. Ends in one of う・く・ぐ・す・つ・ぬ・ぶ・む・る, and that ending shifts across the あいうえお vowel rows when conjugated.
  • Ichidan (一段) — "one-row." Always ends in る; conjugating just means drop る, add the form. The easiest group.
  • Irregular (不規則) — only two exist: する (to do) and 来る／くる (to come). Memorise them.
Careful: some godan verbs also end in る (like 走る, to run), so る alone doesn't prove ichidan. The reliable test: look at the polite form — 〜ります = godan, plain 〜ます = ichidan.⁵
書く (kaku, to write) → godan (ends in く).   見る (miru, to see) → ichidan (る, and polite form is 見ます).   来る (kuru, to come) → irregular (one of the famous two).

🎉 Your tangible win

Go back to the project's README. Its very first verb example is:

type 話すます形 = ConjugateVerb<話す, "ます形">; // 話し

You can now read every piece: 話す is a godan verb type (stem 話 + ending す); ConjugateVerb<…> is a type-level function; "ます形" is the form you asked for; and 話し is what the compiler computed. That's lesson 1. 🌸
